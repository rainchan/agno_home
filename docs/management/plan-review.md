# 计划维护与评审机制

## 工具与跟踪
- 使用 Jira 或 Trello 管理 Epic/Story/Task 与状态流转。
- 看板：Todo/In Progress/Review/Testing/Done，实时更新进度与阻塞项。

## 文档更新
- 任务完成后更新测试报告（覆盖率/性能/结果）、技术文档（接口/架构/设计）、计划调整说明。
- 追溯关系：关联 Story/用例/提交哈希/发布标签。

## 周/迭代评审
- 议题：进度与风险、质量指标、计划优化与建议库筛选。
- 记录：会议纪要、行动项与负责人、截止时间；同步到建议库与计划文档。

## 会议记录模板
- 主题/日期/参会人员/议题清单/讨论要点/结论与评分/决策人/行动项与负责人/截止时间/关联文档链接/备注。

## 变更管理
- 变更申请包含原因/影响/风险/回退方案；评审通过后执行并记录。
- 变更与建议库联动：暂存→评审→纳入版本；保留审计与追溯。
