# 建议库（暂存与评审）

- 目的
  - 暂存所有改进建议与优化方案，待 PRD 与技术架构正式评审通过后，按既定流程筛选与执行
- 范围
  - 产品功能改进、架构优化、性能与安全提升、测试与工具链完善、运维与文档改进
- 生命周期
  - 状态：`暂存` → `评审中` → `已纳入版本`/`已搁置` → `开发中` → `已完成`
- 使用方式
  - 新增建议请以 `suggestion-template.md` 录入，并提交到本目录；并更新 `index.json`
  - 评审结论与变更记录维护在 `changelog.md` 与每条建议的记录中
- 流程对齐
  - 按 `process.md` 的五阶段规范执行：需求确认 → 开发计划制定 → 建议筛选机制 → 执行控制 → 文档管理
- 追溯
  - 每条建议需关联 `PRD`、`技术架构方案`、`版本号/里程碑` 与对应测试用例/实现提交
