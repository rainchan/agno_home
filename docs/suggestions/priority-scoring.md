# 建议优先级评分标准

- 维度与取值（1-5 分）
  - Impact（影响/收益）：对业务价值、用户体验、技术健康度的提升
  - Effort（实现成本）：人天与复杂度；分数越高表示成本越大
  - Risk（风险）：交付与稳定性风险；分数越高表示风险越大
  - Alignment（匹配度）：与当前版本目标、PRD与架构方案的一致性
- 计算建议
  - 标准化得分：`PriorityScore = 0.4*Impact + 0.3*Alignment - 0.2*Effort - 0.1*Risk`
- 映射规则
  - Score ≥ 3.5 → P0（优先纳入当前版本）
  - 3.0 ≤ Score < 3.5 → P1（评审后择机纳入）
  - Score < 3.0 → P2（暂缓，保留在建议库）
- 评审注意
  - 重大安全/合规项无条件优先（P0），独立于打分
  - 需要明确数据与证据支撑（性能指标、缺陷统计、用户反馈）
