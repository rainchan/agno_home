# 建议执行流程（五阶段）

## 1. 需求确认阶段
- PRD 已获得相关方签字确认（产品、设计、运营、业务负责人）
- 技术方案通过架构评审委员会审核（记录评审结论与问题清单）
- 建立版本控制记录（版本号、目标、里程碑、代码分支策略）

## 2. 开发计划制定
- 基于确认后的 PRD 与架构方案拆解任务，制定排期
- 明确阶段里程碑与交付物（文档、代码、测试报告、部署包）
- 评估建议与当前版本匹配度（见 priority-scoring.md），形成纳入/暂缓清单

## 3. 建议筛选机制
- 建立优先级评估标准与评分表（Impact/Effort/Risk/Alignment）
- 组织跨部门评审会议：参与人、议题、结论、行动项与责任人
- 将符合当前版本规划的建议纳入开发计划并更新状态为 `已纳入版本`

## 4. 执行控制
- 严格按确认的开发计划执行，维护燃尽与进度
- 建立变更控制流程（变更申请→评审→批准→记录），关联 PRD/架构方案更新
- 所有修改均需对应需求与方案支撑，提交记录包含链接与说明

## 5. 文档管理
- 维护完整的建议跟踪记录（index.json 与建议文档）
- 确保所有决策有书面依据（会议纪要、审批记录、评审结论）
- 定期更新项目状态报告（周报/月报），含建议处理进度与版本风险
