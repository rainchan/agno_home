# 代码质量标准与评审流程

## 标准
- 代码风格：统一 formatter 与 lint，命名清晰、模块边界明确。
- 错误处理：可恢复策略与统一异常封装；日志不含敏感信息。
- 安全：权限最小化、密钥与 Token 不入库与日志、输入输出校验。
- 文档：接口/架构/设计变更同步；PR 必附测试说明与影响评估。

## 静态分析
- 启用 lint/类型检查/安全扫描；违反规则阻断合并。
- 报告纳入 CI 门禁并归档。

## 评审流程
- PR 模板：变更动机、实现摘要、风险与回退、测试与回归影响。
- 双人 Review 与 CODEOWNERS 强制审批；仅在 CI 全绿后允许合并。
- 变更记录：提交遵循 Conventional Commits；关联 Story/用例与发布标签。

## 覆盖率与门禁
- 核心模块≥90%，非核心≥80%；未达标拒绝合并。
- 与测试报告联动，输出低覆盖文件的补测清单。
